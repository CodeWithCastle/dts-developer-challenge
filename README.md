# 🏛️ HMCTS Task Management System

A simple full-stack task management application designed for administrative workflows. This project features a **React** frontend, a **Node/Express** REST API, and a **PostgreSQL** database, specifically optimized for bulk operations, performance, and data consistency.

## 🌟 Key Features

### Core Functionality

* **Full CRUD Lifecycle:** Create, Read, Update, and Delete tasks with real-time database synchronization.
* **Dynamic Filtering:** Instant search with category views (Todo, In-Progress Done, Overdue).
* **Smart Sideview UI:** GitHub-inspired dual-purpose drawer for both task creation and detailed inspection.
* **Dirty State Protection:** Prevents accidental loss of unsaved changes.

### Performance & Scalability

* **Bulk Operations:** High-performance batch updates and deletes using PostgreSQL `ANY()`.
* **Optimized Rendering:** React performance tuning via `useCallback`, `useContext` and `useMemo`.
* **N+1 Query Elimination:** Refactored inefficient loops into single SQL statements.

### Data Integrity

* **Schema Versioning:** Controlled database initialization using a `schema_info` table.
* **Automated Seeding:** Ensures consistent environments across development and deployment.
* **Sample Seeding:** For test and QA.

## 🏗️ Technical Architecture

### Frontend

* **Framework:** React
* **State Management:** Context API
* **Performance Hooks:** `useCallback`, `useState`, `useEffect`, `useMemo`, `useContext`
* **Design Pattern:** Service abstraction (`taskService`)

### Backend

* **Runtime:** Node.js
* **Framework:** Express
* **Architecture:** RESTful API with structured route hierarchy
* **Key Consideration:** Route ordering to prevent parameter shadowing (e.g., `/bulk` vs `/:id`)

### Database

* **System:** PostgreSQL
* **Design:** Relational schema
* **Optimization:** Bulk queries using `ANY()` operator

## 🚀 Getting Started

### 1. Prerequisites

Make sure you have the following installed:

* [Node.js](https://nodejs.org/) (v16+)
* [PostgreSQL](https://www.postgresql.org/) (running on port 5432)

---

### 2. One-Command Installation

Run the following command from the root directory:

```bash
# Install dependencies across all layers
npm install && cd server && npm install && cd ../client && npm install && cd ..

# Create the database (requires psql CLI)
psql -c "CREATE DATABASE hmcts_tasks;"
```

---

### 3. Environment Setup

#### Server (`/server/.env`)

```env
PORT=8080
DB_USER=dan_aremu
DB_PASSWORD=
DB_NAME=hmcts_tasks
DB_HOST=localhost
DB_PORT=5432
SCHEMA_VERSION=1.0.0
```

#### Client (`/client/.env`)

```env
REACT_APP_API_URL=http://localhost:8080/api/tasks
```

---

### 4. Running the Application

You will need **two terminal sessions**:

#### Terminal 1 — Backend

```bash
cd server
npm run dev
```

* Runs with Nodemon
* Automatically executes `db/init.js`
* Creates tables and seeds data for the current year (2026)

#### Terminal 2 — Frontend

```bash
cd client
npm start
```

## 🛠️ Development Approach

### 1. Database Design

* Implemented a **relational schema**
* Added a `schema_info` table to:

  * Track schema version
  * Prevent duplicate seeding

### 2. Service Layer Abstraction

* Introduced `taskService` on:

  * **Frontend:** API communication abstraction
  * **Backend:** Business logic separation
* Result: Cleaner, maintainable codebase

### 3. API Route Optimization

* Carefully ordered routes:

  * Specific routes (`/bulk`) before dynamic (`/:id`)
* Prevents incorrect route matching and bugs

### 4. Performance Engineering

* Eliminated **N+1 query problems**
* Replaced iterative DB calls with:

  ```sql
  UPDATE tasks SET status = 'done' WHERE id = ANY($1)
  ```
* React optimizations:

  * `useCallback` to memoize functions
  * Reduced unnecessary re-renders

### 5. UX Enhancements

* Built a **Sideview component** for:

  * Task creation
  * Task inspection
* Implemented **Dirty State tracking**:

  * Warns users before losing unsaved changes
